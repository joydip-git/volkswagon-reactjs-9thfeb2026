> npm init --y

transpiler: transformer + compiler
convert ES6+ code into <ES6+ version code for browser backward compatibility
JS => babel compiler

Babel:
---------------
Babel compilet uses a series of "transformers" to convert ES6+ code into <ES6+ version code (vanilla JS)
such as a transformer to convert let keyword into var keyword

plugins: individual package for individual transformer
presets: a package which is a collection of certain plugins

core packages: @babel/core
CLI package: @babel/cli
plugins/presets: @babel/preset-env 


"npm run" CLI command is configured to run a script in package.json file
if in that script any other script/executable name is mentioned, npm will automatically search for it in ".\node_modules\.bin" path. Hence while writing the script no need to provide full path of the executable
for example:  "transpile": "babel src --out-dir=dist"

Modules in JS:
--------------------
an API to expose members of a JS file

module formats:
-----------------------
the format through which we export members of a JS file and also the way we import those exported members in another file

1. Common JS: (the default module format of Node JS)
	a. export: module.exports={prop1:member1, prop2:member2}
	b. import: const {prop1, prop2,..} = require('module-path-w/o-extension')

2. Universal Module Definition (UMD) format:
	a. export: require('',[],function(){ members})
	b. 

3. Asynchronous Module Definition (AMD) format:
	a. export:
	b. import:

4. ES6 module format:
	a. export: export member
	b. import: import {prop1, prop2,...} from 'module-path'
